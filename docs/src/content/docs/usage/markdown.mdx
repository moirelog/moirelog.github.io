Apple Notes 的 富文本格式比较复杂，Moire 支持将其转换为少量 Markdown 语法，以下是功能清单。

![note-example](markdown-support.png)

## Moire 已支持

### 文本格式

| 语法 | 效果 |
|------|------|
| `**粗体**` | **粗体** |
| `*斜体*` | *斜体* |
| `~~删除线~~` | ~~删除线~~ |


### 标题

```markdown
## 二级标题
### 三级标题
```

是的，只支持 H2 和 H3，分别对应 Notes 的 `Title` 和 `Subtitle`。因为 Notes 中的 `Subheading` 实际与 body 大小一致，无法有效区分。

### 链接

```markdown
[链接文本](https://example.com)
```

效果：[链接文本](https://example.com)

### 列表

**无序列表**：
```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
```

**有序列表**：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 引用

````markdown
> 这是一个引用
> 可以用来强调重要内容
````

效果：
> 这是一个引用
> 可以用来强调重要内容


### 图片

```markdown
![图片描述](image-url.jpg)
```

:::note[图片上传限制]
每篇笔记最多支持 1 张图片上传，详见[图片上传机制](/usage/images/)。
:::

## 最佳实践

### 1. 保持简单

使用基础 Markdown 语法，避免依赖高级功能。

### 2. 善用列表

列表比大段文字更易读，优点：

- 简洁清晰
- 易于扫描

### 3. 适当配图

一张图胜过千言万语，但不要贪多（限制 1 张）。
