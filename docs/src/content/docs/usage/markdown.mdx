Apple Notes 的 富文本格式非常复杂，Moire 支持将其转换为少量 Markdown 语法，以下是功能清单。

## Moire 已支持

### 文本格式

| 语法 | 效果 |
|------|------|
| `**粗体**` | **粗体** |
| `*斜体*` | *斜体* |


### 标题

```markdown
# 一级标题
```

是的，只支持 H1。

### 链接

```markdown
[链接文本](https://example.com)
```

效果：[链接文本](https://example.com)

### 列表

**无序列表**：
```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
```

**有序列表**：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 代码块

````markdown
```javascript
function hello() {
  console.log("Hello Moire!");
}
```
````


### 图片

```markdown
![图片描述](image-url.jpg)
```

:::note[图片上传限制]
每篇笔记最多支持 1 张图片上传，详见[图片上传机制](/usage/images/)。
:::

## 最佳实践

### 1. 保持简单

使用基础 Markdown 语法，避免依赖高级功能。

### 2. 善用列表

列表比大段文字更易读：

```markdown
优点：
- 简洁清晰
- 易于扫描
```

### 3. 适当配图

一张图胜过千言万语，但不要贪多（限制 1 张）。

## 测试你的 Markdown

不确定效果？试试这个流程：

<Steps>

1. 在 Moire 文件夹写一篇测试笔记
2. 使用你想测试的语法
3. 同步到网站
4. 查看渲染效果

</Steps>
