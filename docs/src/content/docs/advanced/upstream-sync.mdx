由于 Moire 是通过 **Fork** 方式部署的，当你想要获取页面相关的新功能或修复 Bug 时，需要手动同步上游代码。快捷指令相关内容更新请忽略。

## 方法一：使用 GitHub 网页版（推荐）

这是最简单的方法，不需要使用命令行。

<Steps>

1. 打开你的 GitHub 仓库页面（例如 `yourname/moirelog.github.io`）。

2. 找到文件列表上方的 **Sync fork** 按钮。

3. 点击 **Update branch**。

</Steps>

<Aside type="tip">
如果按钮显示 "This branch is up to date"，说明你的代码已经是处于最新状态。
</Aside>

## 方法二：使用命令行

如果你熟悉 Git，也可以在本地进行合并。

<Steps>

1. 添加上游仓库：
   ```bash
   git remote add upstream https://github.com/moirelog/moire.git
   ```

2. 拉取更新：
   ```bash
   git fetch upstream
   ```

3. 合并到本地主分支：
   ```bash
   git checkout main
   git merge upstream/main
   ```

4. 推送到你的仓库：
   ```bash
   git push origin main
   ```

</Steps>

## 注意事项

- **配置文件**：更新通常不会修改你的 `moire.config.ts`，但如果遇到冲突，请小心处理。
- **自定义修改**：如果你修改了系统核心代码（如页面布局），合并时可能会产生冲突，需要手动解决。
